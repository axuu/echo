# Product

## Register

product

## Users

中文母语用户，在 Windows / macOS 桌面端使用。核心人群是「为了学习、研究或追内容而大量看 B 站 / YouTube / 本地视频的人」：UP 主、学生、研究者、知识工作者。

他们的处境：视频又长又多，没时间逐帧看完（产品口号「懒得看视频？一键省流」）。他们想要的是把视频变成可读、可检索、可追问的文字资产，而不是一直挂在播放器前。隐私敏感——数据要落在本地，不上传云端。技术上愿意也能配置 LLM API Key / 本地 Whisper，但希望整个过程足够傻瓜、足够安心。

要完成的事：贴一个链接（或导入本地视频）→ 转写 → 文本笔记 / 图文笔记 / 思维导图 → 沉淀进知识库 → 之后随时检索与 RAG 问答。

## Product Purpose

Echo 把 B 站 / YouTube / 本地视频，沉淀成可检索、可追问、可导出的本地知识笔记。一条流水线：转写 → 总结 → 图文笔记（含 VLM 理解型）→ 思维导图 → 知识库检索问答，数据全部落本地。深度优化 B 站体验（扫码登录、分 P 处理、风控兜底），同时支持 YouTube 与本地媒体。

成功的样子：用户不再需要看完整段视频，就能拿到一份结构化、配图、可追问的笔记；并且随着看的视频越多，本地知识库越值钱。界面要在「长耗时的 AI 任务」里始终让人安心，在「看笔记」时把内容本身顶到台前。

## Brand Personality

三个词：**轻松友好、活泼、精致有质感**。

声音是中文口语、带点 B 站的俏皮（「一键省流」「把时间留给真正有质量的视频」），但绝不轻浮——它要处理 API Key、费用、本地数据和长耗时任务，可靠和精确不能让位给可爱。情绪目标：第一眼亲切不端着；长任务进行时让人放心（状态、进度、可回溯）；看成品笔记时有「贵」「考究」的质感。粉色（B 站 #fb7299）是识别锚点，但温度主要靠排版、留白、微交互和配图承载，不是靠到处铺粉。

## Anti-references

*(从品牌方向推断，未经用户逐条确认，后续可修正)*

- **冷感的企业级后台 / 行政管理面板**：灰扑扑、密密麻麻、没有温度——和「轻松友好」相悖。
- **千篇一律的 SaaS Dashboard 套路**：大数字 hero 指标模板、等大的 icon+标题+正文卡片网格无限重复。
- **「功能强大」式的信息过载**：把所有开关一次性摊开，吓退非技术用户。
- **游戏化霓虹 / RGB 炫光**：和「精致克制」相悖。

## Design Principles

1. **内容是主角，界面退后。** 转写、笔记、思维导图、知识库内容本身是产品价值所在；chrome（导航、卡片、装饰）要给内容让路，不和内容抢注意力。
2. **长任务要让人安心。** 转写/总结/VLM 合成都是数十秒到数分钟的异步任务。系统状态、进度、可回溯历史、失败可恢复，是这个产品的生命线，不是可选项。
3. **亲切，但永远可靠。** B 站式的俏皮语气是加分项，但它管着密钥、费用、本地数据——任何时候清晰、准确、可预期都优先于「可爱」。
4. **渐进披露复杂度。** 这是一个有很多设置（多提供商、视觉模型独立配置、截图策略、Prompt 预设）的深度工具。当下需要的露出，其余收起，别一次性砸给用户。
5. **一致胜过惊喜。** 同一种操作在任何页面长得一样、行为一样。惊喜留给「时刻」（完成、空状态、首次引导），不是铺满每一屏。

## Accessibility & Inclusion

- 中文优先；正文需达 WCAG AA 对比度（正文 ≥4.5:1，大字 ≥3:1），尤其注意浅灰文字落在浅色背景上的失败。
- 已有 light / dark 双主题，需保证两套主题都达标。
- 需要完整键盘可达性与可见焦点态（桌面工具，重度用户会用键盘）。
- 需要 `prefers-reduced-motion` 兜底，所有动效都要有降级方案。
- 桌面为主，但已有响应式 / 移动布局，触控目标 ≥44×44。
